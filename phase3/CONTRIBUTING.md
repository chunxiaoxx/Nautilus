# 贡献指南 / Contributing Guide

感谢您对 Nautilus 项目的关注！我们欢迎所有形式的贡献。

Thank you for your interest in contributing to Nautilus! We welcome all forms of contributions.

## 如何贡献 / How to Contribute

### 报告问题 / Reporting Issues

如果您发现了 bug 或有功能建议，请：
1. 检查是否已有相关 issue
2. 创建新 issue 并提供详细信息
3. 使用清晰的标题和描述

If you find a bug or have a feature suggestion:
1. Check if there's already a related issue
2. Create a new issue with detailed information
3. Use a clear title and description

### 提交代码 / Submitting Code

1. **Fork 仓库**
2. **创建分支**: `git checkout -b feature/your-feature-name`
3. **编写代码**: 遵循项目代码风格
4. **测试**: 确保所有测试通过
5. **提交**: 使用清晰的 commit message
6. **推送**: `git push origin feature/your-feature-name`
7. **创建 Pull Request**

### 代码规范 / Code Standards

**Python (后端)**
- 遵循 PEP 8
- 使用类型注解
- 编写文档字符串

**TypeScript (前端)**
- 使用 ESLint 配置
- 遵循 React 最佳实践
- 使用函数式组件

### Commit Message 规范

使用语义化提交信息：
- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式
- `refactor:` 重构
- `test:` 测试
- `chore:` 构建/工具

示例：`feat: 添加任务筛选功能`

### 开发流程 / Development Workflow

1. 确保本地环境配置正确
2. 运行测试：`pytest` (后端) / `npm test` (前端)
3. 检查代码风格：`flake8` / `npm run lint`
4. 提交前运行完整测试套件

## 行为准则 / Code of Conduct

- 尊重所有贡献者
- 保持友好和专业
- 接受建设性批评
- 关注项目最佳利益

## 需要帮助？ / Need Help?

- 查看 [README.md](README.md)
- 访问 [API 文档](https://nautilus.social/docs)
- 创建 issue 提问
