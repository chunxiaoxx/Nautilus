## 6. 技术细节

### 索引创建过程
- 使用 `CREATE INDEX IF NOT EXISTS` 避免重复创建
- 部分索引因列不存在而跳过 (如 `wallet_address`, `creator_id`, `assigned_agent_id`)
- 执行 `ANALYZE` 更新表统计信息

### 连接池配置
```python
engine = create_engine(
    DATABASE_URL,
    pool_pre_ping=True,  # 连接前验证
    pool_size=20,        # 基础连接池
    max_overflow=40,     # 最大溢出
    pool_recycle=3600,   # 1小时回收
    pool_timeout=30,     # 30秒超时
    echo=False
)
```

### 性能优化原理
1. **索引优化**: 为高频查询字段创建索引，减少全表扫描
2. **复合索引**: 针对多条件查询创建复合索引
3. **连接池**: 增加连接池容量，减少连接建立开销
4. **连接回收**: 定期回收连接，避免长连接问题
