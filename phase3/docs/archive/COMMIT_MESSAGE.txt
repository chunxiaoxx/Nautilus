feat: Implement Nautilus OAuth 2.0 Service (Task #35)

Implemented a complete, production-ready OAuth 2.0 service for Nautilus Agent
authentication, enabling third-party applications to integrate "Login with Nautilus"
functionality.

## Features

### Core OAuth 2.0 Implementation
- Authorization Code Grant flow
- Token refresh mechanism
- Token revocation support
- Scope-based access control (profile, tasks, balance)
- State parameter for CSRF protection
- Redirect URI validation

### Security
- Client secret hashing (SHA-256)
- Authorization code expiry (10 minutes)
- Access token expiry (1 hour)
- Refresh token expiry (30 days)
- Token revocation support
- Comprehensive input validation

### API Endpoints
- POST /oauth/clients - Register OAuth client
- GET /oauth/clients/{client_id} - Get client details
- GET /oauth/authorize - Authorization endpoint
- POST /oauth/token - Token endpoint (code exchange & refresh)
- GET /oauth/userinfo - Get agent information
- POST /oauth/verify - Verify token validity
- POST /oauth/revoke - Revoke tokens

### Python SDK
- Complete OAuth client implementation
- Easy integration with Flask, FastAPI, CLI apps
- Automatic token refresh
- Comprehensive examples
