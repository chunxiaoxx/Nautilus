## 5. 主要挑战与解决方案

### 5.1 挑战一：模型输出解析与 ThinkingBlock 处理
- **问题**：MiniMax M2.7 模型输出包含 ThinkingBlock，导致 JSON 解析失败和数据提取错误。
- **解决方案**：在 `llm/client.py` 中修复了 ThinkingBlock 处理逻辑，`_extract_text` 和 `_extract_json` 方法现在能够正确跳过思考过程，准确提取最终结果。

### 5.2 挑战二：代码库中的敏感信息清理
- **问题**：`openclaw.json` 等配置文件中包含敏感的 API keys（如 Google、MiniMax、Manus 等），直接开源存在安全风险。
- **解决方案**：已将代码清理列为高优先级任务，计划移除所有真实 keys，并提供 `config.example.json` 示例文件。

### 5.3 挑战三：视频内容制作瓶颈
- **问题**：缺乏图生视频工具（如 Runway/Pika），导致规划的视频内容无法直接生成。
- **缓解措施**：正在寻找合适的图生视频工具或关键帧描述生成方案，以推进视频制作进度。
