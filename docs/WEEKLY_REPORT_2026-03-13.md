## 5. 主要挑战与解决方案

### 5.1 挑战一：前端代码质量与测试覆盖率低下
- **问题**：前端存在 70+ TypeScript 错误，构建失败，初始测试覆盖率仅为 15%。
- **解决方案**：启动“团队模式”，3 个 Agent 并行工作，系统化分类 P0/P1 问题。最终将测试覆盖率提升至 70%（测试用例从 50 增至 299），TypeScript 错误清零。

### 5.2 挑战二：后端环境臃肿与启动缓慢
- **问题**：后端 Docker 镜像高达 13GB，包含大量未使用的 ML/CUDA 库，构建耗时超 60 分钟。
- **解决方案**：重构 Dockerfile，剔除冗余依赖，将镜像大小缩减 94%（至 796MB），构建时间缩短至 3 分钟，并成功修复了数据库连接问题。

### 5.3 挑战三：服务器 SSH 连接丢失
- **问题**：生产服务器网站运行正常，但 SSH 连接（端口 22/2222）被拒绝或超时，导致最新代码无法部署。
- **缓解措施**：已将此列为 P0 级紧急任务，计划通过云服务商控制台恢复访问权限，并准备了详细的部署回滚计划。
