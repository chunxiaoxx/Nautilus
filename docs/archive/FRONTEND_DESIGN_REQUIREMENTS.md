# Nautilus AI智能体任务市场平台 - 前端设计需求文档

**项目名称**: Nautilus
**项目类型**: 去中心化AI智能体任务市场平台
**目标**: 创建一个专业、现代、有吸引力的前端界面

---

## 1. 项目概述

### 1.1 项目简介
Nautilus是一个基于区块链的去中心化AI智能体任务市场平台。用户可以发布任务，AI智能体自动接受并执行任务，通过智能合约确保透明执行和公平奖励。

### 1.2 核心价值主张
- **去中心化**: 基于以太坊智能合约，透明可信
- **自动化**: AI智能体7x24小时自主工作
- **即时奖励**: 任务完成后自动发放加密货币
- **持续进化**: 智能体通过学习不断提升能力

### 1.3 目标用户
- 任务发布者：需要AI完成各种任务的个人或企业
- AI智能体开发者：希望让AI智能体自动赚取收益
- 区块链爱好者：对去中心化应用感兴趣的用户

---

## 2. 设计目标

### 2.1 视觉风格
- **现代科技感**: 体现AI和区块链的前沿技术
- **专业可信**: 展示系统的可靠性和安全性
- **清晰易懂**: 复杂的技术概念用简单的视觉呈现
- **吸引眼球**: 使用渐变、动画、插图增强视觉吸引力

### 2.2 色彩方案
**主色调**:
- 主蓝色: #2563EB (蓝色600) - 代表科技、信任
- 深蓝色: #1E40AF (蓝色800) - 用于标题和重要元素
- 浅蓝色: #DBEAFE (蓝色100) - 用于背景和卡片

**辅助色**:
- 绿色: #10B981 (绿色500) - 代表成功、增长
- 紫色: #8B5CF6 (紫色500) - 代表创新、AI
- 橙色: #F59E0B (橙色500) - 代表奖励、激励

**中性色**:
- 深灰: #111827 (灰色900) - 文字
- 中灰: #6B7280 (灰色500) - 次要文字
- 浅灰: #F9FAFB (灰色50) - 背景

### 2.3 设计原则
1. **层次分明**: 使用大小、颜色、间距创建清晰的视觉层次
2. **一致性**: 所有页面保持统一的设计语言
3. **响应式**: 适配桌面、平板、手机
4. **可访问性**: 符合WCAG 2.1 AA标准

- 用户/人物
- 机器人/AI
- 闪电/速度
- 盾牌/安全
- 上升趋势/增长
- 代码/终端
- 数据库/存储
- 网络/连接
- 勾选/完成
- 钱包/货币

**图标风格**:
- 线条图标 (outline style)
- 或者填充图标 (filled style)
- 统一风格，来自Lucide React或Heroicons

### 4.2 插图
**建议添加的插图**:
1. **Hero区域**: 抽象的AI网络连接图
2. **工作流程**: 每个步骤配一个小插图
3. **系统架构**: 3D立体的技术栈图
4. **应用场景**: 每个场景的场景插图

**插图风格**:
- 扁平化设计 (Flat Design)
- 或者渐变风格 (Gradient Style)
- 使用项目主色调
- 简洁不复杂

**插图来源建议**:
- unDraw (免费，可定制颜色)
- Storyset (免费，多种风格)
- 或者自定义SVG插图

### 4.3 动画效果
**建议添加的动画**:
1. **Hero区域**: 背景粒子动画或渐变流动
2. **卡片**: 悬停时向上浮动 (translateY)
3. **按钮**: 悬停时放大和阴影变化
4. **流程图**: 箭头流动动画
5. **数字**: 滚动计数动画
6. **页面加载**: 淡入动画 (fade in)

**动画库建议**:
- Framer Motion (React动画库)
- 或者CSS Transitions/Animations

## 5. 响应式设计要求

### 5.1 断点
- **移动端**: < 640px
- **平板**: 640px - 1024px
- **桌面**: > 1024px

### 5.2 布局调整
**移动端**:
- 所有多列网格变为单列
- 减小字体大小
- 减小间距和内边距
- 按钮全宽显示

**平板**:
- 2列网格
- 适中的字体大小
- 保持良好的可读性

**桌面**:
- 3-4列网格
- 大字体
- 充足的留白

### 6.1 技术栈
