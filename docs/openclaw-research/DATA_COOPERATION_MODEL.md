# 数据合作模式 - 章鱼动力 × Nautilus

## 背景

Nautilus = 数字世界的执行大脑（System 2/1/0 分层架构）
章鱼动力 SYNTH = 物理世界的执行大脑（感知→决策→执行→验证闭环）

两者在 **数据层面** 的合作，是形成「数字+物理闭环」的关键。

---

## 数据合作模式

### 模式一：任务下放（Task Handoff）

**章鱼动力 → Nautilus**
```
物理世界任务 → 数字世界子任务 → 结果回传 → 物理执行
```

| 方向 | 数据类型 | 说明 |
|------|----------|------|
| 下行 | 任务描述 + 约束条件 | 物理任务拆分后的数字子任务 |
| 上行 | 执行结果 + 验证报告 | 子任务完成状态 |

**典型场景：**
- 章鱼机器人需要某个复杂任务的最优解 → 发给 Nautilus 推理
- 物理任务涉及大量数据查询/计算 → Nautilus 完成计算，结果传回

**数据量：** 任务描述（< 10KB）→ 执行结果（< 1MB）

---

### 模式二：数据即服务（Data-as-a-Service）

**双向数据共享**

```
章鱼动力提供：         Nautilus 提供：
- 物理传感器数据        - 数字世界知识图谱
- 机器人状态数据        - 用户行为画像
- 环境感知数据          - 任务执行历史
- 操作反馈数据          - LLM 推理结果
```

**接口形式：**

```python
# 章鱼 → Nautilus 数据 API
POST /v1/data/physical
{
    "source": "synth_robot_01",
    "data_type": "sensor_reading | robot_state | environment",
    "payload": {...},
    "timestamp": "2026-03-22T10:00:00Z"
}

# Nautilus → 章鱼 结果 API
POST /v1/data/digital
{
    "source": "nautilus",
    "data_type": "inference_result | action_plan | verification",
    "payload": {...},
    "timestamp": "2026-03-22T10:00:00Z"
}
```

---

### 模式三：联邦学习（Federated Learning）

**数据不出本地，模型协同训练**

```
章鱼动力（物理数据）─────┐
                        ├──→ 联合模型训练 ──→ 共享模型
Nautilus（数字数据）─────┘
```

**适合场景：**
- 任务执行策略优化（物理+数字数据联合训练）
- 机器人泛化能力提升
- 个性化推荐（结合物理位置+数字偏好）

**数据安全：** 原始数据不离开本地，只有梯度/参数交换

## 商业价值量化

| 合作方向 | 对章鱼动力价值 | 对 Nautilus 价值 |
|----------|---------------|-----------------|
| 任务下放 | 复杂计算外包，降低机器人算力需求 | 获取真实物理世界任务数据 |
| 数据共享 | 丰富训练语料，提升机器人泛化 | 扩展数字世界感知边界 |
| 联合优化 | 端到端任务成功率提升 | 形成差异化：唯一有物理出口的数字 Agent |

## Demo 设计（Phase 1 成果展示）

**场景：服务机器人复杂任务规划**

```
用户："帮我把会议室的投影仪打开"

章鱼机器人：
  感知：会议室位置已知，但投影仪品牌/型号未知
  ↓ 发给 Nautilus
Nautilus：
  查询知识图谱：会议室设备清单
  LLM 推理：投影仪开机流程（品牌无关）
  返回：标准操作序列
  ↓ 回传给章鱼
章鱼机器人：
  执行：打开投影仪 → 切换 HDMI → 调焦
  验证：确认投影画面正常
```

**价值：** Nautilus 补强了章鱼机器人的"知识盲区"，使其不需要内置所有设备操作手册。
